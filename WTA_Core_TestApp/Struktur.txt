Beschreibung der Struktur
Core
Config

ConfigLoader.cs: Singleton-Klasse zum Laden und Speichern von Konfigurationen.
DefaultConfig.cs: Stellt Standardkonfigurationen bereit.
ConsoleHelper

Menu.cs: Stellt eine Menüstruktur zur Verfügung.
TableRenderer.cs: Rendert Tabellen in der Konsole.
Interfaces

IBaseModel.cs: Interface für grundlegende Modelle mit einer ID.
IConfigLoader.cs: Interface für Konfigurationsloader.
IGenericRepository.cs: Interface für generische Repositorys.
ILogger.cs: Interface für Logger.
ILogObserver.cs: Interface für Log-Observer.
ITestBase.cs: Interface für Testbasisklassen.
Logging

ConsoleLogObserver.cs: Beobachter für Konsolenlogs.
Logger.cs: Singleton-Klasse für das Logging.
Models

BaseModel.cs: Abstrakte Basisklasse für Modelle mit einer ID.
TestModel.cs: Beispielmodellklasse.
Event.cs: Modellklasse für Events.
Repositories

GenericRepository.cs: Generisches Repository für CRUD-Operationen.
TestCase

ConfigLoaderTest.cs: Testklasse für ConfigLoader.
GenericRepositoryTest.cs: Testklasse für GenericRepository.
LoggerTest.cs: Testklasse für Logger.
MenuTest.cs: Testklasse für Menu.
TableTest.cs: Testklasse für TableRenderer.
TestBase.cs: Abstrakte Basisklasse für Tests.
Utilities

ConfigHelper.cs: Stellt Hilfsfunktionen für Konfigurationen bereit.
LogHelper.cs: Stellt Hilfsfunktionen für Logging bereit.
EventHelper.cs: Stellt Hilfsfunktionen für Events bereit.
WTA_Console
Models

Worker.cs: Modellklasse für Mitarbeiter.
WorkTime.cs: Modellklasse für Arbeitszeiten.
Utilities

ConsoleHelper.cs: Hilfsklassen für Benutzereingaben in der Konsole.
WorkerHelper.cs: Stellt Hilfsfunktionen für Worker bereit.
WorkTimeHelper.cs: Stellt Hilfsfunktionen für WorkTime bereit.
WorkTimeManager.cs: Verwaltet die Arbeitszeiten.
WorkerManager.cs: Verwaltet die Mitarbeiter.
EventManager.cs: Verwaltet die Events.
Apps

TestMenuConsole.cs: Stellt ein Testmenü für Konsolenanwendungen bereit.
